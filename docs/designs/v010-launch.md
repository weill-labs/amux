## Vision

### 10x Check
The 10x version of amux isn't "tmux with more features" — it's the terminal that
agents use natively. The agent API (structured JSON, blocking waits, push events) is
already unlike anything tmux offers. The 10x play is making that API so good that
Claude Code, Codex, and Cursor ship with amux integration built in. That's the
post-v0.1.0 story — v0.1.0 is the credibility signal that enables it.

### Strategic Context (as of 2026-03-20T21:00Z)
- **Project age:** 7 days (first commit 2026-03-13)
- **Velocity:** ~260 commits, ~280 PRs total (262 merged at snapshot time), 82 Linear issues completed
- **Codebase:** ~48K lines Go, ~58% test-to-code ratio, 16 packages
- **Development model:** 9 parallel Codex agents + 1 human
- **12-month goal:** Homebrew-installable tmux replacement (~5K GitHub stars)
- **Strategic approach:** Ship Early, Iterate with Users (Approach B)

### Key Insight
The biggest risk isn't missing features — it's building the wrong features. 9 agents
generating work for each other with no external user signal creates an echo chamber.
Shipping v0.1.0 breaks the cycle and introduces real feedback.

## Flake Severity Definitions
- **P1:** Blocks CI pipeline (always fails, or fails >50% of runs)
- **P2:** Fails >10% of CI runs — unreliable signal for contributors
- **P3:** Fails <10% of runs — annoying but not blocking
- **P4:** Theoretical or unreproduced — observed once, not confirmed

## No-Go Criteria
- CI has P1/P2 flakes that reproduce on GitHub Actions
- PR #252 requires major rework (not just rebase/minor fixes)
- `brew install` flow fails on macOS arm64 or amd64
- Linux amd64 binary fails to start or pass smoke test (test in CI or Docker)
- Blog post isn't human-edited before posting

## Rollback Plan
If a critical bug surfaces within 24h of release: tag `v0.1.1` with the fix,
update the Homebrew formula, and post an update comment on the HN thread.
`go install @latest` picks up the fix automatically.

## Launch Prerequisites (v0.1.0)
1. Merge PR #252 (v0.1.0 prep: CHANGELOG, benchmarks, demo, API contract, comparison — also delivers `docs/comparison.md`)
2. Resolve all open PRs — human triages: merge if CI-green and review-approved, close with "deferred post-v0.1.0" label otherwise
3. CI green — zero P1/P2 known flakes (P3/P4 flakes may be tolerated with skip annotations)
4. Homebrew tap works: `brew install weill-labs/tap/amux`
   - (a) Create `weill-labs/homebrew-tap` GitHub repo
   - (b) Add `HOMEBREW_TAP_GITHUB_TOKEN` secret to the amux repo
   - (c) Add `brews:` stanza to `.goreleaser.yaml`
   - (d) Test with `goreleaser release --snapshot --clean`
5. README includes: one-line pitch, architecture diagram, install instructions, agent API quick-start, comparison link
6. Blog post (~1500 words, markdown, published on project GitHub Pages or personal blog) drafted by agent, human-edited, ready for launch day
7. tmux compatibility checklist published in `docs/comparison.md` (delivered by PR #252, may need enhancement)

## Post-Launch Plan
- Reduce from 9 agents to 3-4, focused on user-reported issues
- Add `amux doctor` diagnostic command
- Use XDG_RUNTIME_DIR for socket paths on Linux
- Verify crash recovery covers hot-reload failure path
- Verify minimum pane size guards (1x1 resize, over-split)
- Add end-to-end smoke test (start → split → type → capture → detach → reattach → verify)
